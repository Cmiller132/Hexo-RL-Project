# Documentation Index

This directory keeps the current, useful documentation at the top level.

Older research notes, one-off audits, dated handoffs, and superseded implementation plans have been moved to:

```text
Docs/archive/
```

## Current top-level docs

- `GETTING_STARTED.md`: quick project setup and orientation.
- `SYSTEM_DESIGN.md`: current high-level system architecture.
- `TRAINING.md`: training workflow notes.
- `RUST_API.md`: Rust-facing API notes.
- `game.md`: game rules and domain reference.
- `OPTUNA_SEQUENTIAL_AUTOTUNING_PLAN.md`: current Optuna/autotuning plan.
- `GLOBAL_GRAPH_PAIR_HEADS.md`: current global graph pair-head notes.
- `RGSC_IMPLEMENTATION.md`: RGSC implementation notes.
- `sources.md`: source/reference list.

## Refactor docs

Docs/refactor/

## Archive guidance

Use `Docs/archive/` for dated or superseded documents that may still be useful for historical context but should not be treated as current implementation guidance.
